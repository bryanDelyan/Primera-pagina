Definición del problema

Dentro del paradigma estructurado la definición del problema es lo que generalmente se conoce como Análisis.

en esta fase se realiza la Planeación del proyecto y se determinan los requerimientos del sistema. Esta planeación incluye las necesidades que el cliente ha expresado, las necesidades internas de la organización y principalmente la misma misión de la empresa.

Lo primero que se debe de hacer para la planeación de un proyecto es definir un glosario con todas las terminologías que utiliza el cliente, también se debe de describir a grandes rasgos cual es el principal problema, la solución planeada, y las restricciones de las soluciones antes propuestas. Hay que tomar en cuenta que no porque el usuario haya expresado cuáles son las respuestas que quiere que le de el sistema,

los desarrolladores no tengan que tomar encuenta ciertos aspectos de un tipo más técnico como los algoritmos que mejor resuelvan esos problemas, o el tipo de base de datos que se vaya a utilizar.

Es muy importante que el encargado de la planeación del proyecto , se involucre con el cliente y observe detenidamente como se llevan a cabo los procesos y procedimientos de la organización, puede utilizar diversas técnicas para lograr identificar los problemas de manera correcta.

Otro aspecto importante a considerar son los recursos que vamos a utilizar para desarrollar el sistema, como el lenguaje de programación y el tipo de hardware. Este tipo de consideraciones incluyen también los costos, tanto de hardware y software como el costo de desarrollar el sistema, que obviamente debe de ser razonable para que el cliente lo acepte.

Según Fairley en su libro sobre Ingeniería de Software menciona que para definir un problema es necesario:

Desarrollar un enunciado definitivo del problema por resolver. Incluir una descripción del sistema actual, restricciones del problema y de las metas que se lograran. El enunciado del problema debe de realizarse empleando terminología del cliente.

Justificar la estrategia de solución computarizada. Identificar las funciones por realizar, las restricciones, el subsistema del equipo electrónico, el subsistema del producto de programación, y del personal. Determinar los objetivos y requisitos en el nivel del sistema para el proceso de desarrollo y los productos finales.Establecer los criterios de alto nivel para la aceptación del sistema.

Fairley también menciona algunos de los factores que se deben de considerar para la planeación de un proyecto:

Técnicas de estimación que se utilizarán; precisión requerida.

Modelo del ciclo de vida, funciones de control y revisiones.

Estructura organizacional.

Nivel de formalidad en especificaciones, planes de prueba, etc.

Nivel de verificación y validación

Nivel de administración de la configuración requerida.

Nivel de control de calidad requerido.

Responsabilidades de seguimiento y mantenimiento

Herramientas que se desarrollan y emplearán.
